异星工厂服务器搭建

发表于 2018-12-27 | 更新于 2018-12-28

《异星工厂》（Factorio）为Wube Software所开发的一套冒险沙盒游戏。发布有Microsoft Windows、MacOS与Linux平台，2016年在Steam平台上发布。 目前Steam平台售价90元，好评率98%，而且官方声明无任何打折或降价的计划。
故事基线以一名宇航员（也就是玩家）的飞船坠落到一个未知的星球上，为了能再度重返地球，玩家必须利用星球表面的所有可用资源，建立各种生产设备，升级科技，最终发射载有卫星的火箭通关游戏。与此同时，生产设备所产生的污染也会吸引星球的原著生物攻击，必须建立适当的防御来抵挡攻击。
为方便与同伴进行联机游戏，游戏商提供了服务器端程序，玩家可以建立联机服务器来进行联机游戏，提高游戏乐趣。下面将介绍在搭载Linux系统服务器上假设Factorio联机服务器的教程。
服务器系统：CentOS 7 x64
服务器端Factorio版本：0.16.51

安装 glibc-2.18

curl -O http://ftp.gnu.org/gnu/glibc/glibc-2.18.tar.gz
tar zxf glibc-2.18.tar.gz
cd glibc-2.18/
mkdir build
cd build/
../configure --prefix=/usr
make -j2
make install

下载服务器端程序

wget https://www.factorio.com/get-download/0.16.51/headless/linux64

解压程序

tar -xvf linux64
解压完成后得到文件夹factorio，其目录结构如下：
.
+——bin
+——config-path.cfg
+——data

新建存档文件夹

mkdir saves

#修改服务器端配置文件
cd /factorio/data
vim server-settings.example.json
配置文件注释如下所示

将配置文件重命名为 server-settings.json

mv server-settings.example.json server-settings.json

将已有的存档上传至saves文件夹或在服务器上生成新存档

随机创建新存档
./bin/x64/factorio --create ./saves/map.zip

加载存档，运行服务端

nohup ./bin/x64/factorio --start-server /home/factorio/saves/map.zip &
回车，输入exit退出shell

开放游戏端口(默认为34197)

/sbin/iptables -I INPUT -p tcp --dport 34197 -j ACCEPT
/etc/rc.d/init.d/iptables save
service iptables restart

查看服务器端运行日志

cat /home/factorio/bin/x64/nohup.out
正常日志状况

进入游戏客户端，点击多人游戏→服务器直联
输入xx.xx.xx.xx:34197（xx.xx.xx.xx为服务器的ip地址，端口默认为34197）
